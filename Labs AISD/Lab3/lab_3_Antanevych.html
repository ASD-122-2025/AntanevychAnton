Визначення асимптотичної складності алгоритмів сортування¶

  • Сортування вставками (Insertion Sort): O(n²)
  • Сортування бульбашкою (Bubble Sort): O(n²)

Це означає, що зі збільшенням розміру вхідних даних (n) у 2 рази, час виконання зростає приблизно в 4 рази.

# Алгоритм сортування вставками:
def insertion_sort(nums):
    for j in range(1, len(nums)):
        key = nums[j]
        i = j - 1
        while i >= 0 and nums[i] > key:
            nums[i + 1] = nums[i]
            i -= 1
        nums[i + 1] = key

# Приклад
arr = [5, 3, 8, 1, 2]
insertion_sort(arr)
print(arr)        

[1, 2, 3, 5, 8]

# Алгоритм сортування бульбашкою
def bubble_sort(nums):
    n = len(nums)
    for i in range(n):
        for j in range(0, n - i - 1):
            if nums[j] > nums[j + 1]:
                nums[j], nums[j + 1] = nums[j + 1], nums[j]

# Приклад
arr = [5, 3, 8, 1, 2]
bubble_sort(arr)
print(arr)

[1, 2, 3, 5, 8]

def measure_time(sort_func, data):
    start_time = time.time()
    sort_func(data.copy())
    return time.time() - start_time

Контрольні питання¶

  1. Що таке O-нотація і чим вона відрізняється від Ω-нотації?

    • O-нотація (Big-O) описує верхню межу зростання часу або ресурсів, які споживає алгоритм у найгіршому випадку.
    • Ω-нотація (Big-Omega) вказує на нижню межу — тобто мінімальний час виконання в найкращому випадку.
    • Вони відрізняються напрямком оцінки: O() — "не більше ніж", Ω() — "не менше ніж".

  2. Яку часову складність має алгоритм сортування методом вставляння за найсприятливіших умов, коли масив відсортовано? Відповідь обґрунтувати.

    • У найкращому випадку, коли масив вже відсортований, Insertion Sort має лінійну складність O(n),
      оскільки кожен елемент просто порівнюється з попереднім і не пересувається.

  3. Надати визначення ефективного алгоритму.

    • Ефективний алгоритм — це такий, що розв’язує задачу за розумний (поліноміальний) час і з мінімальними витратами ресурсів (пам’яті, енергії тощо),
      особливо при зростанні розміру вхідних даних.

  4. Пояснити поняття головного параметра (розміру) завдання.

    • Головний параметр — це змінна, яка характеризує розмір вхідних даних.
      Наприклад, для масиву це кількість елементів n, для графа — кількість вершин або ребер.

  5. Функція часової складності має вигляд: F(N) = N³ + 7N² − 14N. Як записати асимптотичну складність у O-нотації?

    • При великому N найважливішим є доданок з найвищим степенем — N³.
      Отже, асимптотична складність: O(N³).
