  1. Оцінка асимптотичної складності алгоритмів
Алгоритм Складність Пояснення
Лінійний пошук O(n) Перевіряє всі елементи послідовно у найгіршому випадку
Бінарний пошук O(log n) На кожному кроці ділить масив навпіл, швидко звужуючи область пошуку

import matplotlib.pyplot as plt

# 1. Визначаємо функції пошуку
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

def measure_time(search_func, arr_size, search_func_name):
    arr = list(range(arr_size))

  1. Висновки про ефективність алгоритмів
  • Бінарний пошук демонструє значно кращу продуктивність для масивів розміром > 100 елементів
  • Лінійний пошук може бути ефективнішим для дуже малих масивів (n < 10) через відсутність накладних витрат
  • Для відсортованих даних бінарний пошук завжди є оптимальним вибором
  • При необхідності частого пошуку варто використовувати бінарний пошук після попереднього сортування даних

Оптимальний вибір алгоритму залежить від:

  1. Частоти пошукових операцій
  2. Необхідності підтримки відсортованого стану даних
  3. Очікуваного розміру вхідних даних
№ Контрольне питання Відповідь
1 Сформулювати задачу пошуку Задача пошуку полягає у знаходженні позиції елемента з певним значенням у заданій структурі даних (масиві, списку, дереві тощо) або підтвердженні його відсутності.
2 Які критерії можна використовувати для оцінювання ефективності алгоритмів пошуку?
  • Часова складність (найгірший/середній випадок)
  • Використання пам'яті
  • Необхідність попередньої обробки даних
  • Адаптивність до вхідних даних
4 Що таке лінійний пошук і яка його асимптотична складність? Лінійний пошук — це перевірка кожного елемента послідовно. Складність: O(n) у найгіршому випадку.
5 Що таке бінарний пошук і в чому полягає його головний принцип роботи? Бінарний пошук — алгоритм для відсортованих масивів, що ділить область пошуку навпіл на кожному кроці. Принцип: порівняння з елементом у середині та відкидання непотрібної половини.
6 Які переваги і недоліки бінарного пошуку порівняно з лінійним? Переваги:
  • Швидкість (O(log n))
Недоліки:
  • Вимагає відсортованих даних
  • Складніша реалізація
7 Які ще алгоритми пошуку існують, окрім лінійного і бінарного?
  • Пошук стрибками (Jump Search)
  • Інтерполяційний пошук
  • Експоненційний пошук
  • Пошук у хеш-таблицях (O(1))
  • Пошук у деревах (BST, AVL, B-дерева)
